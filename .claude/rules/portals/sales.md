# 영업사원 포털 규칙

> 영업사원(GET) 포털의 탭 구조, 인센티브 정책, KPI 공식

---

## 🎯 포털 개요

**역할**: 영업사원은 고객을 관리하고 판매·리뉴얼 활동을 수행하며, 인센티브와 KPI 달성률로 평가됨

**디렉토리**: `doc/sales/` (또는 `doc/get/`)

**메뉴 ID**: `S01`, `S02`, `S03` ... (또는 `G01`, `G02` ...)

---

## 📑 탭 구조 및 기능

### 1. 대시보드 (dashboard.php)
**메뉴 ID**: S01

**이번 달 주요 지표**:

- 리뉴얼 수
- 유지 고객
- 보고 건수

**KPI 달성률 게이지**: NORMAL / WARNING / GOAL ACHIEVED

### 3. 리뉴얼 관리 (renewal_mgmt.php)
**메뉴 ID**: S03

**계약 만료 30일 이내** 고객 자동 표시

**리뉴얼 인센티브 자동 계산**:

- 연속 리뉴얼: 40,000원

**프로세스**: 리뉴얼 요청 → HQ 승인 → 자동 반영

**자동 계산**:

## 💰 인센티브 정책

### 판매 인센티브
- **총액**: 90,000원/대
- **지급 방식**: 15,000원 × 6회 분할 지급
- **조건**: 계약 유지 시에만 지급
- **해지 시**: 잔여 미도래분 자동 소멸
- **지급일**: 익월 15일

### 리뉴얼 인센티브
- **기본**: 30,000원
- **연속 리뉴얼**: 40,000원
- **지급 시기**: 결제 확인 후 익월 15일 지급

### 예상 수입 계산
```javascript
이번 달 예상 수입 = 분할 예정 합 + 리뉴얼 예정 합 + 활동비
```

## 📊 KPI 공식

### KPI 구성 비율
```
KPI = 판매(40%) + 유지(25%) + 리뉴얼(20%) + 보고(15%)
```

### 세부 계산식
```javascript
// 판매율
salesRate = (thisMonthNewSales / targetSales) * 100;

// 유지율
retentionRate = (activeCustomers / totalCustomers) * 100;

// 리뉴얼율
renewalRate = (renewedCustomers / renewalTargets) * 100;

// 보고율
reportRate = (submittedReports / targetReports) * 100;
```

### KPI 점수 계산
```javascript
KPI점수 = (판매율×0.4 + 유지율×0.25 + 리뉴얼율×0.2 + 보고율×0.15) × 100
```

---

## 📐 자동 계산 공식

```javascript
// 1. 월 예상 인센티브
monthlyIncentive = (salesCount × 15000) + (renewalCount × 30000) + activityFee;

// 2. 판매 인센티브 분할
salesIncentiveInstallment = salesCount × 15000; // 6회 분할 중 이번 달

// 3. 리뉴얼 인센티브
renewalIncentive = renewalCount × (isConsecutive ? 40000 : 30000);

// 4. KPI 점수
kpiScore = (
  (salesRate × 0.4) +
  (retentionRate × 0.25) +
  (renewalRate × 0.2) +
  (reportRate × 0.15)
) * 100;

// 5. KPI 배지 표시
kpiBadge = kpiScore >= 80 ? 'ACHIEVED' : (kpiScore >= 60 ? 'NORMAL' : 'WARNING');
```

## 🏷️ 상태 배지

### 계약 상태
- **ACTIVE**: 초록
- **WARNING**: 노랑
- **TERMINATED**: 회색

### KPI 배지
- **ACHIEVED** (80% 이상): 초록
- **NORMAL** (60~79%): 파랑
- **WARNING** (60% 미만): 노랑
